# План: новая надстройка VZID

## Этап 1. Согласование продукта

- Зафиксировать целевой пользовательский сценарий.
- Подтвердить способ обновления.
- Подтвердить целевой интерфейс.
- Подтвердить стратегию хранения версий и отката.
- Подтвердить, какие старые сценарии остаются, а какие убираются.

## Этап 2. Архитектурное проектирование

- Спроектировать загрузчик.
- Спроектировать структуру основной надстройки.
- Спроектировать схему релизов в GitHub.
- Спроектировать хранение настроек пользователя.
- Спроектировать схему логирования и диагностики.

## Этап 3. Подготовка репозитория

- Создать структуру нового проекта.
- Зафиксировать единый публичный репозиторий как источник кода и релизов.
- Описать правила сборки и выпуска.
- Подготовить документы для разработчика и для публикации релизов.

## Этап 4. Реализация MVP

- Базовый загрузчик.
- Базовая вкладка `VZID`.
- Выбор региона.
- Несколько ключевых макросов как эталонные команды.
- Проверка обновления через GitHub Releases.

## Этап 5. Миграция функций

- Перенос макросов по группам.
- Упрощение или замена старых `UserForm`.
- Проверка региональных различий.

## Этап 6. Подготовка к запуску

- Подпись.
- Проверка на корпоративных машинах.
- Проверка сценариев обновления и отката.
- Подготовка инструкции для пользователей.
