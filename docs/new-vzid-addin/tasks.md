# Задачи: новая надстройка VZID

## Текущий статус

- `[согласовано]` Продуктовая спецификация согласована на базовом уровне.
- `[согласовано]` Основной интерфейс: вкладка `VZID` на ленте Excel.
- `[согласовано]` Базовая схема обновления через GitHub Releases утверждена.
- `[согласовано]` Базовая роль загрузчика утверждена.
- `[согласовано]` Базовая стратегия миграции старых сценариев утверждена.
- `[ожидает]` После согласования начать новую кодовую базу.

## Первые обязательные решения

- `[частично согласовано]` Источник релизов: GitHub Releases как основной источник.
- `[согласовано]` Один общий пакет для всех регионов.
- `[согласовано]` Режим обновления: мягкое уведомление с кнопкой обновить.
- `[согласовано]` Обновление применяется только после полного закрытия Excel и нового запуска.
- `[согласовано]` После скачивания обновления пользователю предлагается перезапустить Excel сразу или отложить это действие.
- `[согласовано]` Основной интерфейс: вкладка ленты `VZID`.
- `[согласовано]` Хранение пользовательских настроек должно включать запоминание региона.
- `[согласовано]` Главный выбор региона переносится из `frmTerritory` на вкладку `VZID`, остальные действующие `UserForm` по умолчанию сохраняются.
- `[согласовано]` Учитывать установку без прав администратора.
- `[согласовано]` Первый входной способ установки: один `setup.exe`.
- `[согласовано]` Учесть разграничение доступа по Windows-логину.
- `[согласовано]` Кнопки без доступа должны быть видимы, но неактивны.
- `[согласовано]` Правила доступа хранятся в конфиге надстройки.
- `[согласовано]` Если GitHub недоступен, показывать маленький alert на вкладке `VZID`, не блокируя работу.
- `[согласовано]` Предусмотреть резервную установку обновления из локального файла.
- `[согласовано]` Исходный код можно держать приватно, а релизный репозиторий сделать публичным.
- `[согласовано]` Первый MVP должен покрывать пилотные команды для Кургана, Тюмени и Екатеринбурга.
