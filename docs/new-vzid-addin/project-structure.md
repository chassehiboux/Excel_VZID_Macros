# Структура проекта: новая надстройка VZID

## Статус

Рабочий документ для старта реализации MVP.

## Что это даст в работе

В репозитории сразу будет понятное разделение:

- где лежит загрузчик;
- где лежит основная бизнес-надстройка;
- где лежат шаблоны конфига и релиза;
- где лежит документация для сборки и выпуска.

Это нужно, чтобы дальше не собирать проект из случайных кусков и чтобы публикация новой версии была предсказуемой.

## Базовое решение по репозиторию

- Репозиторий один: `https://github.com/chassehiboux/Excel_VZID_Macros`
- Он же хранит исходники, документацию, установщик и GitHub Releases.
- Загрузчик проверяет обновления по `manifest.json` из `latest release`.

Базовый URL для MVP:

- `https://github.com/chassehiboux/Excel_VZID_Macros/releases/latest/download/manifest.json`

## Зоны ответственности

### `installer/`

Содержит описание состава `setup.exe` и правила первой установки без прав администратора.

### `release/`

Содержит шаблон `manifest.json` и договорённость по именам файлов релиза.

### `config/`

Содержит шаблон пользовательского `config.json`, который установщик кладёт в профиль пользователя.

### `src/vzid-loader/`

Содержит исходники маленького загрузчика.

Задача загрузчика в MVP:

- создать и проверить рабочие папки;
- прочитать базовый конфиг;
- подключить активную рабочую версию `MainVZID`;
- проверить наличие новой версии `MainVZID`;
- скачать обновление в пользовательский профиль;
- пометить его как ожидающее активации после полного закрытия Excel.

Ограничение MVP:

- автообновление самого загрузчика пока не делаем;
- новый загрузчик ставится через `setup.exe` или через ручную замену релизного файла.

### `src/main-vzid/`

Содержит исходники основной надстройки.

Задача основной надстройки в MVP:

- показать вкладку `VZID`;
- отрисовать dropdown выбора региона;
- показать общий набор кнопок;
- отключать недоступные команды;
- маршрутизировать кнопку в нужный макрос по выбранному региону;
- показывать статус обновления и сервисные действия.

## Модули загрузчика

- `LoaderEntryPoint.bas`
  точка входа при запуске Excel.
- `LoaderPaths.bas`
  пути `%LocalAppData%\\VZID\\...`.
- `LoaderConfig.bas`
  чтение и запись базовых настроек.
- `LoaderAddinHost.bas`
  подключение и отключение `MainVZID`.
- `LoaderUpdates.bas`
  проверка `manifest.json`, подготовка ожидающего обновления.
- `LoaderLogging.bas`
  простой лог загрузчика.

## Модули основной надстройки

- `MainConfig.bas`
  чтение и запись общего `config.json`.
- `MainEntryPoint.bas`
  общий вход и сервисные команды.
- `RibbonVZID.bas`
  callbacks ленты Excel.
- `RegionState.bas`
  текущий регион и его сохранение.
- `CommandRegistry.bas`
  список бизнес-команд и их привязка к макросам по регионам.
- `CommandDispatcher.bas`
  запуск нужного макроса по выбранной кнопке.
- `AccessPolicy.bas`
  права по Windows-логину.
- `UpdateStatus.bas`
  чтение статуса обновления для отображения на вкладке.

## Состав релиза MVP

Файлы, которые должны лежать в GitHub Release:

- `MainVZID.xlam`
- `manifest.json`
- `setup.exe`

Файлы, которые можно добавить позже:

- `LoaderVZID.xlam`
- `release-notes.md`

## Договорённость по manifest

`manifest.json` в MVP должен описывать:

- версию релиза;
- дату публикации;
- URL скачивания `MainVZID.xlam`;
- `sha256` файла `MainVZID.xlam`;
- минимальную поддерживаемую версию загрузчика;
- ссылку на заметки к релизу.

## Договорённость по config

`config.json` в MVP должен хранить:

- выбранный регион;
- текущую активную версию `MainVZID`;
- путь к ожидающему обновлению;
- время последней проверки обновлений;
- статус последней проверки;
- правила доступа по логину;
- логин `dzuikevich` с полным доступом по умолчанию.

Техническое упрощение MVP:

- пока используем плоские строковые поля JSON;
- права доступа по командам храним в `...Csv`-полях;
- это упрощает первый стабильный прототип без тяжелого JSON-парсера в VBA.

## Что делаем следующим шагом

1. Импортировать этот каркас в тестовые `LoaderVZID.xlam` и `MainVZID.xlam`.
2. Привязать к новой ленте реальные макросы и оставшиеся рабочие `UserForm`.
3. Доделать скачивание релиза и проверку `sha256`.
4. Собрать первый пользовательский `setup.exe`.
