# Архитектура: новая надстройка VZID

## Статус

Базовая архитектура после согласования продуктовых решений.

## Общая схема

Продукт состоит из трёх частей:

1. `Setup`
2. `Loader`
3. `MainVZID`

## 1. Setup

### Назначение

Один `setup.exe`, который пользователь запускает без прав администратора.

### Что делает

- создаёт служебные папки в профиле пользователя;
- кладёт туда загрузчик;
- кладёт стартовый конфиг;
- регистрирует загрузчик как обычную Excel-надстройку пользователя;
- не использует `Program Files`;
- не использует `XLSTART` как точку самоподмены.

### Целевые папки

- `%LocalAppData%\VZID\loader\`
- `%LocalAppData%\VZID\versions\`
- `%LocalAppData%\VZID\config\`
- `%LocalAppData%\VZID\logs\`

## 2. Loader

### Назначение

Маленькая стабильная надстройка, которая запускается вместе с Excel и подключает рабочую версию `MainVZID`.

### Что должен делать

- читать конфиг;
- определять текущую активную версию;
- подключать рабочую надстройку;
- проверять обновления по расписанию;
- показывать статус обновления;
- скачивать новую версию;
- активировать её только после полного закрытия Excel.

### Что не должен делать

- запускать `vbs`, `cmd`, `powershell`;
- самозаменяться через `XLSTART`;
- выполнять тяжёлую бизнес-логику;
- содержать крупные формы и бизнес-макросы.

## 3. MainVZID

### Назначение

Основная надстройка со всей бизнес-логикой.

### Что содержит

- вкладку `VZID`;
- dropdown выбора региона;
- кнопки бизнес-сценариев;
- привязку кнопок к макросам по региону;
- существующие отдельные `UserForm`, где они реально нужны;
- правила доступа к командам;
- сервисные команды.

## Логика интерфейса

### Основные принципы

- у пользователя один вход через вкладку `VZID`;
- регион выбирается в dropdown;
- набор кнопок визуально стабилен;
- выбранный регион управляет доступностью и маршрутизацией команд;
- недоступные команды остаются видимыми, но неактивными.

### Общие кнопки MVP

- `Пакеты`
- `Сопроводительные`
- `Сопроводительные по получателю`
- `Заказные`

### Маршрутизация по регионам

#### Курган

- `Пакеты` -> `Doc_Packets`
- `Сопроводительные` -> `Make_Cover_Letters`
- `Сопроводительные по получателю` -> `Make_Cover_Letters_ByRecipient`

#### Тюмень

- `Пакеты` -> `Doc_Packets`
- `Сопроводительные` -> `TMN_Make_Cover_Letters_likeKGN`
- `Сопроводительные по получателю` -> `TMN_Make_Cover_Letters_ByRecipient_likeKGN`

#### Екатеринбург

- `Пакеты` -> `Doc_Packets`
- `Заказные` -> `RIC_Zakaznye_Create`

## Существующие формы

### Что убирается

- `frmTerritory` как главный экран выбора региона.

### Что заменяется вкладкой

- стартовая логика региональных экранов:
  `frmVZID_KGN`, `frmVZID_TMN`, `frmVZID_EKB`, `frmVZID_CHLB`.

### Что сохраняется

- прочие сценарии, где отдельное окно уже используется как рабочий инструмент;
- например `ManualProcReport`.

## Обновление

### Основной путь

Источник: GitHub Releases.

Загрузчик:

- читает локальный метаданный файл;
- при необходимости запрашивает актуальный `manifest.json`;
- сравнивает версии;
- скачивает новый `.xlam`;
- проверяет `sha256`;
- сохраняет новую версию как ожидающую активации.

### Резервный путь

Пользователь вручную выбирает файл релиза через кнопку:

- `Установить обновление из файла`.

### Когда применяется обновление

- не в текущей живой сессии Excel;
- только после полного закрытия Excel;
- при следующем полном запуске.

### Что видит пользователь

- компактный alert на вкладке `VZID`;
- кнопку `Обновить`;
- кнопку `Установить обновление из файла`;
- сообщение после скачивания:
  обновление будет применено только после полного закрытия Excel.

## Конфиг

### Путь

- `%LocalAppData%\VZID\config\config.json`

### Что хранит

- выбранный регион;
- интервал проверки обновлений;
- текущую активную версию;
- путь к ожидающей версии;
- правила доступа по Windows-логину;
- служебные флаги интерфейса.

## Права доступа

### Базовая модель

- проверка по Windows-логину;
- правила лежат в конфиге;
- `dzuikevich` имеет максимальные права по умолчанию.

### Поведение интерфейса

- если доступ есть, кнопка активна;
- если доступа нет, кнопка видна, но неактивна;
- при необходимости показывается поясняющая подсказка.

## Антивирусная стратегия

### Главные принципы

- никаких shell-скриптов для подмены файлов;
- никаких скрытых `vbs/cmd/powershell`;
- предсказуемая файловая структура;
- контролируемое скачивание файлов;
- проверка хэша скачанного релиза;
- разделение загрузчика и бизнес-логики;
- цифровая подпись должна быть предусмотрена как отдельный этап.

## Следующий уровень проектирования

После этого документа следующий шаг:

- расписать структуру проекта по папкам;
- определить модули загрузчика;
- определить модули основной надстройки;
- определить формат `manifest.json`;
- определить состав `setup.exe`.
